> 你设计的渐进式披露，对人有效，对 AI 无效。不是 AI 不够聪明——是你把判断的责任放在了错误的位置。

 
## 一套漂亮的设计

 记忆系统的前台存盘 Skill 负责一件事：每次有意义的工作进展，执行三步存盘——写 log（人看）、写 buffer（后台守护进程看）、更新 context（下次开机看）。跳过任何一步都是 bug。

 最初的设计，核心文件只有 74 行。三步存盘的流程写得很清楚。至于防腐检查、中途检查点、搜索历史、巡检——这些都放在 `modules/` 目录下，按需加载。

 设计意图很明确：降低认知噪音。Agent 日常只需要消化 74 行核心逻辑，遇到特定场景再去加载对应模块。渐进式披露，教科书级优雅。

 
## C1/C2 事故

 然后事故来了。

 两次独立的记忆同步遗漏，前后不到一周。我们事后把它们编号为 C1 和 C2。

 表现形式一样：Agent 执行了存盘，log 写了，context 更新了，但 buffer 里的内容格式不对。后台守护进程按约定格式解析 buffer，解析失败，默默跳过。前台以为存了，后台没收到。记忆链断了。

 追根因：buffer 的详细格式规范写在 `modules/` 下。Agent 加载了核心 SKILL.md，看到"写 buffer"，就按自己对"写 buffer"的理解写了一行。那行缺了类型标签，缺了 source 字段。格式不对。

 为什么 Agent 没去加载 `modules/` 里的格式规范？因为 SKILL.md 里确实有一行："详细的 buffer 格式见 modules/memory-sync.md"。但 Agent 没读到——或者读到了、没当回事。74 行里提到了"详细规范见其他文件"，Agent 自认为核心逻辑已经够了。

 **Agent 不知道自己不知道什么。** 按需加载的前提是消费者能判断"现在需要加载那个模块了"。但如果消费者连那个模块的存在意义都不清楚——它凭什么做出这个判断？

 
## 急救

 紧急发布 v3：把 `modules/` 里的关键逻辑全部内联回 SKILL.md。

 存盘协议、buffer 格式、经验分类规则、归档格式——所有和存盘流程直接相关的规则，一个文件写完。74 行变成 200 多行。渐进式披露死了。

 但存盘再也没出过问题。

 `modules/` 目录还在。没人删它，也没人用它。像截肢后的幻肢。

 
## 同构问题在更高层重演

 v3 修了一个 Skill 内部的按需加载问题。但同样的结构性问题在更高一层重演了。

 记忆系统由三个 Skill 组成：一个管初始化（给新项目建记忆目录），一个管前台存盘（就是上面说的那个），一个管后台维护（多级任务——分类、合并、跨项目关联、一致性审计）。三者共享数据结构、协议、概念体系。

 用户说"记忆"，Agent 得知道该加载哪个。如果项目还没有记忆目录——初始化。如果是日常工作——存盘。如果是后台定时任务——维护。

 但 Agent 怎么知道该选哪个？它得先了解这三个 Skill 各管什么。它得先加载"某个东西"来告诉自己该加载哪个 Skill。这是一个鸡生蛋的问题。

 最终方案：三合一。合并为一个 Skill，内部自动判断模式。项目没有记忆目录 → 初始化模式；环境变量标记后台 → 守护模式；其他 → 存盘模式。

 一个入口，三种模式。Agent 不需要判断自己"需要"哪个——加载这一个，模式判断在内部完成。

 
## 结论

 传统软件工程说职责分离。初始化、运行时、后台维护——这三件事放到任何架构教科书里，都应该是不同模块。

 但这里的消费者不是人，是 Agent。

 人会看目录结构，会读 README，会主动搜索"我需要什么"。Agent 不会。Agent 的注意力模型是线性的：它读到什么就用什么，没读到的不存在。它不会主动去"找"该加载哪个模块、该用哪个 Skill。

 按需加载靠的是消费者的自律——看到触发条件就去加载对应模块。人的自律可以通过习惯、培训、code review 来逐步建立。Agent 的"自律"——在文档里写一行"遇到 X 场景请加载 modules/X.md"——本质上是在纯文本里嵌入一条 if-else 分支，然后祈祷 Agent 每次都能精确执行。

 这不是 Agent 不够聪明。这是把"知道自己需要什么"的判断责任交给了一个没有全局视图的消费者。这件事在任何消费者身上都不完全可靠——只不过人可以通过经验逐渐变可靠，而 Agent 每次开机都是新的。

 **当消费者是 AI 的时候，一个入口比职责分离更重要。**

 这个判断有适用条件：Skill 之间确实存在强耦合（共享数据结构和概念体系），且总量在 Agent 上下文窗口的承受范围内。如果两个 Skill 之间没有交集，硬合在一起反而是噪音。

 合并不是万能的。但在"让 AI 做选择"和"替 AI 做选择"之间，后者几乎总是更安全的默认策略。
