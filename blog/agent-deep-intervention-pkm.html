当 Agent 开始管你的笔记

2026-04-28 · 标签: AI Agent, 人机协作, 知识管理

我有一个日记系统，运行了一段时间。每天一个 markdown 文件，记任务、记碎碎念、记突然冒出来的想法。最早用 Obsidian，后来切到 IDE + Agent 工作流之后，Obsidian 的 GUI 反而碍事了，扔掉了，但日记的习惯留了下来。

Agent 介入之后，事情变了。

传统 GTD 系统有个老问题：系统本身需要你花精力去维护。每天打开 Todoist 挪卡片、给任务打标签、清理过期项、回顾 Someday/Maybe 列表——这些维护动作不产出任何成果，但不做就会腐烂。系统越精密，维护成本越高，最后管理任务的时间比做任务还多。

Agent 能把这些全接管。它读得懂你的日记，知道你今天在做什么，昨天搁置了什么，这周的目标还差多远。它不需要你挪卡片、打标签、做回顾——你说一句话，它自己去处理。基本等于有了一个能读懂你的全能秘书，把几乎所有琐碎的维护工作搞定，剩下的就是喂饭给你吃了。

迭代了半个多月，这套系统长出了几个有意思的设计。

3 天 Sprint：Agent 替你盯节奏

每 3 天选 1-2 个核心目标。每天开工时 Agent 自动检查：Sprint 还剩几天、目标进度如何、今天打算做的事跟目标对不对得上。偏了就提醒，但不阻止。你不用操心"我是不是忘了什么"——它帮你记着。

为什么是 3 天？试过不同的周期。1 周太长——AI 时代节奏快，到第 4 天目标可能已经过时了，"这周 OKR 还没完成"的焦虑开始累积。1 天太短——任何需要深度思考的事至少要 2 天才出成果，一天一换等于在水面上打水漂。

跟传统 Scrum Sprint 有个关键区别：不排他。传统 Sprint 期间做 Sprint 外的事是违规的。但个人工作里紧急事务不可预测，完全排他产生的是焦虑不是聚焦。这里的 Sprint 更像车里的导航仪——你可以绕路，它不会抢方向盘，但会显示你偏了多远。

传统 GTD 里这件事叫"每日回顾"——每天花十分钟翻任务列表、重新排优先级。有用，但需要纪律。现在 Agent 自动替你做了。你只管干活，它帮你感知"今天干的事离目标有多远"。

冰箱：Agent 替你管搁置

传统 GTD 的 Someday/Maybe 列表有个通病：变成垃圾场。扔进去就忘了，偶尔翻一下，全是过时的念头，越看越焦虑，然后再也不看了。根本原因是维护成本太高——你得自己去翻、去清理、去判断哪些还值得留。

我管搁置区叫"冰箱"。名字就是设计意图——冰箱里的东西不是垃圾，是你有意保鲜的食材。Agent 自动维护每一项的冻结日期、来源、搁置理由。你不用打标签、不用手动归档——说一句"这个先冰起来"就行。

一开始加了自动激活：冻结超过 7 天自动提醒。听起来很合理？失败了。自动提醒让冰箱变成了另一个焦虑来源——你明明有意搁置的事，系统不停催你。

改成纯被动：只有你主动问"冰箱里有什么"时才汇报。想取回就说一声。Agent 老老实实维护数据，但不主动开口。你搁置一个想法，不需要任何操作成本，也不会产生心理负担。

碎碎念里的 @ai：随时叫出来的深度讨论

日记里的碎碎念什么都有：吐槽、灵感、待办、情绪。大部分只是记录，但偶尔有一条，你觉得"这个值得展开聊聊"——不是让 AI 帮你执行什么，而是想跟它深入讨论一个想法。

标记方式：在末尾加一个 @ai。就这么简单。AI 次日开工时自动扫描，看到 @ai 就主动发起讨论。

这个功能的价值比看起来大得多。传统的笔记系统里，一个灵感从"记下来"到"被认真思考"之间有一道巨大的鸿沟——你得找时间、调状态、重新进入那个语境。大部分灵感就死在这道鸿沟里了。现在 Agent 替你跨过去：你只管记，标个 @ai，它第二天就带着上下文来跟你聊。

试过"所有对话自动沉淀"——让 Agent 自己判断哪些值得保留。噪音太大。"帮我改个格式"、"把这段移到上面"这些日常指令全混进来了。之前的文章里说过：这类场景下，人定接口，Agent 走流程，比反过来靠谱。@ai 就是那个接口——轻到几乎没有成本，但能精确区分"记录"和"讨论"。

回头看这几个设计，它们的共同点不是"帮你做减法"——这个说法太浅了。真正发生的事情是：传统 GTD 里那些需要人来维护的流程摩擦，被 Agent 悄悄接管了。

每日回顾？Agent 自动做。任务归档？说一句话就行。搁置管理？它帮你记着，你忘了也没关系。灵感从"记下来"到"被讨论"？标个 @ai，第二天它自己来找你。

GTD 的理念没问题——收集、整理、回顾、执行。问题一直出在"整理"和"回顾"上，这两步全靠人的纪律，而人的纪律是世界上最不可靠的资源。现在 Agent 把这两步接过去了。你只管收集和执行，中间那些琐碎的维护活儿，它全包了。

这才是革命性的——不是 AI 帮你想出了更好的方法论，而是它把旧方法论里"人做不到"的那部分变成了"自动就做了"。
